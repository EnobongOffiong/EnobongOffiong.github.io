Mizzou Final Exam Scheduler

React Node.js Express AWS REST API GitHub Actions

Description: I created a scalable full-stack application that automated exam date retrieval for over 1,000 students, eliminating their need for manual website searches. The system features a React frontend and a Node.js backend with a dedicated AWS database, which together serve exam schedules through a RESTful API, reducing lookup time by 50%. I automated testing and deployment with GitHub Actions CI/CD, which enhanced release reliability and reduced manual maintenance. This project successfully delivered a reliable and efficient tool that significantly improved student experience.

Project Reflection

"This project really showed me why you shouldn't just jump into coding. Taking the time to plan the backend with diagrams saved me from a ton of problems later on. I got way better at using Express to set up API routes and, honestly, finally figured out how to properly manage database connections without everything crashing. The biggest struggle was making sure the app wouldn't slow down with more users, but getting it to work on AWS was a huge win. It felt pretty awesome knowing something I built actually made a difference for so many students."

Quizzical

React.js JavaScript CSS3 API Integration Responsive Design

Description: Quizzical is a dynamic and engaging quiz web app I built using React.js. It fetches a diverse set of questions from the Open Trivia Database API to provide a unique experience every time. The app features a clean, responsive interface designed with CSS, ensuring it works seamlessly on both desktop and mobile devices. I focused on optimizing performance and user experience by efficiently managing state and side effects with React hooks which minimized unnecessary re-renders and created a smooth, interactive flow from start to finish.

Project Reflection

"This project was my deep dive into really understanding React. I learned that managing state properly is everything - using useState and useEffect the right way totally changed how smooth the app felt. The biggest challenge was figuring out how to structure the API calls and then manage the flow of data without the UI freezing up. It was satisfying to finally get the play again button to work correctly at the end. Seeing it all come together into something that's actually fun to use made all the debugging worth it."
